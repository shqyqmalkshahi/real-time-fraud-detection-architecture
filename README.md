# Real-Time Fraud Detection & AML Compliance Architecture

##  Overview
This project presents a scalable big data architecture designed for real-time fraud detection and anti-money laundering (AML) compliance in the banking sector.

---

##  Problem
Financial institutions must detect fraud in real time while handling high transaction volumes and meeting strict regulatory requirements.

---

##  Architecture Components
- NiFi → Data ingestion
- Kafka → Real-time streaming
- Spark → Fraud detection & analytics
- HDFS → Distributed storage
- Hive → SQL & compliance reporting
- HBase → Real-time data access
- Solr → Search and investigation layer
- YARN → Resource management

---

##  System Design
The system processes transaction streams in real time, applies anomaly detection models, and assigns risk scores to identify fraudulent behavior.

---

##  Business Value
- Real-time fraud detection
- Reduced financial loss
- Improved AML compliance
- Scalable infrastructure for high transaction volumes
